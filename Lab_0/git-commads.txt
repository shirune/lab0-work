1. git clone** - Создает локальную копию удаленного репозитория на вашем компьютере. Это первая команда для начала работы с существующим проектом
2. git status** - Показывает состояние рабочего каталога и области подготовленных файлов: какие файлы изменены, но не добавлены для коммита, а какие уже подготовлены[citation:3].
3. git add** - Добавляет изменения из рабочего каталога в "индекс" (staging area), подготавливая их к следующему коммиту. Можно добавить как конкретный файл (`git add file.txt`), так и все файлы (`git add .`).
4. git commit** - Создает постоянный "снимок" (commit) всех подготовленных командой `git add` изменений и сохраняет его в локальную историю репозитория. Ключ `-m` позволяет добавить сообщение. Например: `git commit -m "Добавлен README.md"`[
5. git push** - Загружает (отправляет) ваши локальные коммиты в удаленный репозиторий (например, на GitHub), делая изменения доступными для других
